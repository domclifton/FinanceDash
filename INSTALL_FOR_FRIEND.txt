INVESTHOME FINANCE TRACKER v3.0.0-rc.1 - EASY INSTALL
=================================================

GitHub install:

   git clone https://github.com/domclifton/InvestHome.git
   cd InvestHome
   chmod +x scripts/install.sh scripts/start.sh scripts/backup_db.sh
   ./scripts/install.sh
   ./scripts/start.sh

Manual install:

   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   python3 app.py

Open it in a browser:

   http://SERVER-IP:5000

TRADING 212 OPTIONAL SETUP
==========================

Optional: add Trading 212 credentials in Settings → Trading 212 Connection after first launch.
Credentials are saved in the application settings and can be overwritten from the same page. Sync creates/updates Trading 212 ISA (Auto) in Account Balances, which has a read-only synced value and is included in totals unless its Type is set to Ignore.

UPDATING FROM AN OLDER VERSION
==============================

Keep these files:

   data/finance.db
   .env

Before updating:

   ./scripts/backup_db.sh

Then overwrite/pull the app files and restart.

RESETTING THE APP
=================

To start from fresh data:

   rm data/finance.db
   python3 app.py

DO NOT COMMIT
=============

Do not commit:

   .env
   data/finance.db

These are already ignored by .gitignore.


Database notes for v3.0.0-rc.1:
- The app stores the live SQLite database at data/finance.db.
- If an older root-level finance.db exists, the app moves it into data/ on first launch.
- Backups, imports, restores and undo-last-action are available from Settings -> Database.

Data/display notes for v3.0.0-rc.1:
- Accounts page Type now includes Ignore. Ignored accounts remain visible but are excluded from dashboard statistics.
- Trading 212 ISA (Auto) value remains managed by the API, but its dashboard Type can be changed locally.
- Property page can include or ignore property equity in total net worth.


UI notes for v3.0.0-rc.1:

- Manual account names can be edited inline from Account Balances.
- Take Snapshot is now in the left sidebar under the All data is synced card.
